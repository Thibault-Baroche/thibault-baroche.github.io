Automatic training log from Strava | Technical part

The achievement of this project had two specific objectives. The first one, as explained on the previous page, was to develop a dashboard that would allow me to delve into my training statistics in a precise manner. The second objective was the implementation of a fully automated process, to familiarize myself with the Zapier tool and focus on the upstream part of the data analysis process: data collection.

Strava is seamlessly integrated into Zapier, greatly simplifying the process. To begin, I created a new Zap following this scheme:

As soon as an activity is added to my Strava account, the trigger is activated, and it adds a new line in an associated Google Sheet file. It retrieves the following fields via the Strava API:

To avoid interfering with the import process that overwrites all formulas with each new added line, the rest of the data cleaning and preparation will be done in another Google Sheet file. In this file, I exported my activity history since 2013 and added it to the first tab. In a second tab, I retrieve all the data from Zapier using the IMPORTRANGE function. This way, I get the raw data as it comes out of the API:

Then, simply create a new tab that combines all the old activities downloaded before automation via Zapier and the newly imported data, format them in the correct formats and units:

From now on, it's just a matter of linking this data sheet with Looker Studio and moving on to the dashboard creation part!

This small project, without major technical challenges, was a good way to create a first automated dashboard, addressing a personal need and making it easier for me to visualize my training statistics.
